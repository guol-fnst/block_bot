# Block Bot - CWS Listing Copy

## 1. 简短描述（<=132 字符）

在 X/Twitter 页面扫描已显示推文，或对指定博主进行深度扫描，识别疑似垃圾/机器人账号，并在你确认后本地分批执行屏蔽。

## 2. 详细描述

Block Bot 帮你在 X（Twitter）上更高效地清理垃圾账号。

核心功能一：单页快速扫描
在当前页面一键发起分析：扩展会读取你已看到的推文内容，发送到你配置的模型服务（如 Gemini、DeepSeek、Qwen）进行判断。结果会以列表形式展示账号、置信度和原文，让你一眼看清。

核心功能二：目标博主深度扫描（Deep Scan）
输入任何一个博主 Handle，自动在其最近的帖子中翻页采集评论，并对比不同用户的回复相似度。轻松识别并一网打尽那些“复制粘贴”协同作弊的引流机器人。扫描在后台自动进行，切换标签页或关闭弹窗后重新打开，进度仍会保留。

安全与限速屏蔽：
所有屏蔽操作都需要你明确勾选并确认。扩展会按安全限速策略在你的本地浏览器中分批处理，必要时自动处理网络错误及超时重试，并在遇到账号封禁时自动触发熔断。

隐私与数据处理说明：
- 完全无后端：Block Bot 不会将任何数据上传到自有服务器。
- 数据仅在你的浏览器本地处理，直接发往你选择的模型服务商 API。
- API Key 仅加密保存在你的本地浏览器存储中。

适用场景：
- 批量识别可疑营销号、机器人号、刷屏及引流诈骗账号。
- 在保留人工绝对控制权的前提下，数十倍提升屏蔽效率。

## 3. 隐私权页面（必填项文案）

### 3.1 单一用途说明
本扩展的单一用途是：在用户当前打开的 X/Twitter 页面中识别疑似垃圾或机器人账号，并在用户明确确认后执行本地屏蔽操作。

### 3.2 请求 storage 的理由
用于在本地保存用户配置（模型提供商、模型名称、API Key、阈值与同意状态）。这些数据仅保存在用户浏览器本地，不会上传到 Block Bot 服务器。

### 3.3 请求 tabs 的理由
用于读取当前活动标签页 URL，判断是否为 x.com / twitter.com，并将分析与屏蔽任务绑定到当前标签页。

### 3.4 请求 scripting 的理由
用于在 x.com / twitter.com 页面注入并执行内容脚本，以读取用户当前可见推文并在用户确认后执行页面内屏蔽交互。

### 3.5 请求主机权限的理由
仅请求 x.com 与 twitter.com 主机权限，用于在这些页面运行内容脚本，完成可见内容采集与用户确认后的屏蔽操作；扩展不会在其他网站运行。

### 3.6 远程代码选项
建议选择：不，我并未使用远程代码。

说明：扩展会向用户自行配置的第三方模型 API 发起 HTTPS 请求以获取分析结果，但不会下载并执行远程 JS/Wasm 代码。
