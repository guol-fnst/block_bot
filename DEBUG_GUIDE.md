# 深度扫描调试指南

## 问题：深度扫描显示"未找到疑似账号"

如果你确实看到了可疑账号，但插件没有检测到，请按以下步骤调试：

### 步骤 1：打开浏览器开发者工具

1. 在 Chrome 中打开插件管理页面：`chrome://extensions/`
2. 找到 "Block Bot" 扩展
3. 点击"Background pages"（背景页）下的链接打开 Service Worker 的开发者工具

### 步骤 2：运行深度扫描并查看日志

1. 在插件 popup 中点击"深度扫描某博主"
2. 输入博主 handle、配置参数
3. 点击"开始扫描"
4. 保持开发者工具窗口打开
5. 在 Console 选项卡中观察日志输出

### 步骤 3：理解日志输出

日志格式示例：
```
[DeepScan] 开始扫描 @example, 配置: maxPosts=20, maxRepliesPerPost=100, maxTotalReplies=1000
[DeepScan] 采集到 15 条帖子链接
[DeepScan] 第 1 条帖子采集到 25 条回复
[DeepScan] 累计回复数: 25
...
[DeepScan] 原始回复数: 145, 排除OP后: 144
[DeepScan] 本地预过滤 - 明显机器人: 5, 需要模型分析: 139
[DeepScan] 模型分析 - 返回结果数: 139
[DeepScan] 自动回复检测后 - 总结果数: 144
[Normalize] 收到结果总数: 144, 置信度门槛: 0.8
  [0] @account1 (displayName) 置信度: 0.95 ✓ ✓ 是机器人
  [1] @account2 (displayName) 置信度: 0.65 ✗ ✗ 不是机器人
  ...
[Normalize] 过滤后通过门槛的: 12
```

### 关键指标解释

1. **采集到的帖子数** - 如果为 0，说明没有找到任何帖子
2. **采集到的回复数** - 如果为 0，说明帖子没有回复（或采集失败）
3. **本地预过滤** - 检测明显的机器人（keyword、随机handle、emoji 等）
4. **模型分析** - 使用 LLM 分析的结果数
5. **置信度门槛** - 默认为 0.8（80%）
6. **过滤后的候选人** - 最终符合置信度门槛的账号数

### 常见问题排查

#### 问题 1：采集到的回复为 0
- 帖子可能确实没有回复
- 或者 X.com 的页面结构可能已变化，导致采集失败
- **解决方案**：检查"第 X 条帖子采集到 0 条回复"的日志

#### 问题 2：采集到回复，但本地预过滤全部过滤
- 这不太可能，因为预过滤非常严格（需要多个条件同时满足）
- **解决方案**：查看是否有"本地预过滤 - 明显机器人"的数量

#### 问题 3：LLM 分析返回数量太少或置信度都很低
- LLM 模型的判断可能不准确
- **解决方案**：
  1. 检查"置信度"列中的具体数值
  2. 如果大部分都在 0.6-0.79 范围内，尝试降低置信度门槛（在设置页）
  3. 检查自定义检测提示词是否有效（在设置 → 检测提示词）

#### 问题 4：符合门槛的候选人数少于预期
- 原因可能是置信度门槛设得太高（默认 0.8）
- **解决方案**：
  1. 在选项页面降低置信度门槛（如改为 0.7）
  2. 或者调整自定义检测提示词以提高 LLM 的敏感度

### 步骤 4：调整参数

根据日志找到的问题，尝试以下调整：

1. **降低置信度门槛**：
   - 打开扩展选项页面
   - 在"垃圾邮件置信度门槛"中改为更低的值（如 0.65 或 0.7）

2. **优化检测提示词**：
   - 在选项页面的"检测提示词"中添加你想要检测的特定行为
   - 例如针对特定的营销话术或机器人特征

3. **增加采集范围**：
   - 在深度扫描配置中增加最多帖子数或回复数

### 报告 Bug

如果调整后仍然没有效果，请提供：
1. 博主 handle（需要确认有可疑账号回复）
2. 浏览器开发者工具中的完整日志输出
3. 你认为应该被检测到的具体账号示例
