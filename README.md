# Block Bot

Block Bot 是一个面向 `x.com` / `twitter.com` 的 Chrome 扩展，用来扫描当前页面已显示的推文，识别疑似垃圾账号或机器人账号，并在用户确认后将这些账号加入本地屏蔽队列。

## 当前行为

扩展只在 `x.com` 与 `twitter.com` 页面运行。它会读取当前页面已经渲染出来的推文内容、账号 handle、显示名称、推文链接和资料页链接，然后把候选数据发送到用户在设置页中选择的模型服务进行分类分析。

屏蔽动作不会通过 Block Bot 自己的服务器执行。用户确认后，扩展会在本地浏览器内使用当前登录的 X 账号进行限速屏蔽；如果当前页无法直接完成某一笔操作，扩展可能临时在后台打开一个 X 页面来完成该屏蔽任务。

## 权限说明

- `storage`: 保存模型服务配置、分析缓存和队列状态。
- `tabs`: 查询当前标签页并在必要时创建后台 X 标签页完成屏蔽。
- `activeTab`: 当前版本仍保留，建议上架前移除。
- `scripting`: 向已授权的 X 页面注入脚本以读取页面内容并执行本地自动化。
- `host_permissions` for `x.com` / `twitter.com`: 限定扩展只在这些站点工作。

## 数据流与隐私

扩展当前没有自建后端。候选推文数据会直接发送给用户配置的模型服务提供商，例如 Gemini、DeepSeek、Qwen，或用户自行配置的 OpenAI 兼容接口。

这意味着上架 Chrome Web Store 时，你需要：

1. 提供公开可访问的隐私政策 URL。
2. 在商店后台如实披露会处理的页面内容与账号标识数据。
3. 明确说明数据会发送到用户选择的第三方模型服务。
4. 在商店文案中写明扩展可能在后台打开 X 页面完成本地屏蔽。

仓库内的隐私政策底稿见 `PRIVACY_POLICY.md`。

## 开发

1. 打开 Chrome 的扩展程序页面。
2. 启用开发者模式。
3. 选择“加载已解压的扩展程序”，加载本目录。
4. 打开 X 页面后，从弹窗启动分析。

## 上架前待办

- 移除不必要权限，优先删除 `activeTab`。
- 收窄或白名单化自定义模型服务地址，减少隐私披露复杂度。
- 将 `PRIVACY_POLICY.md` 发布到公开 URL，并填写到 Chrome Web Store。
- 准备商店截图、隐私字段和更精确的功能描述。
